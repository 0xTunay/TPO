# Отчёт по тестированию ПО — Веб-приложение “GitHub”

## 1. Введение
**Цель работы:**  
Провести системное тестирование веб-приложения “Github” для проверки корректности работы функционала, выявления дефектов и оценки соответствия требованиям.  

**Краткое описание программы:**  
Веб сайт позволяет контролировать версии ПО, позволяет заниматься совместной разработкой и разработкой в open source проектах

**Обоснование тестирования:**  
Системное тестирование необходимо для оценки поведения приложения с точки зрения конечного пользователя, проверки соответствия требованиям (SRS), интерфейса и производительности.

---

## 2. Тест-план (Test Plan)

### 2.1 Цели и задачи
- Проверка корректности добавления, редактирования и удаления коммитов  
- Проверка работы отправки PR   
- Проверка корректного отображения readme файла  
- Выявление ошибок и дефектов  
- Проверка соответствия требованиям разработки

### 2.2 Область применения
**Тестируемые модули:**  
- Модуль управления коммитами  
- Модуль управления ветками  
- Модуль Модуль отображения Markdown  
- Модуль GitHub Actions  
- Модуль добавления релизов  

**Ограничения:**  
- Тестирование проводится только на ubuntu-latest  

### 2.3 Стратегия тестирования
- Функциональное тестирование всех модулей  
- Регрессионное тестирование после исправления багов  
- Тестирование GUI для проверки корректного отображения страницы репозитория, Pull Requests, отображение README  
- Smoke-тестирование авторизация, открытие репозитория, просмотр файлов, создание коммита через web-интерфейс  

### 2.4 Критерии начала тестирования
- Пользователь может успешно пройти аутентификацию
- Работают базовые операции: создание файла, commit, создание ветки
- Доступны основные разделы: репозитории, Issues, Pull Requests

### 2.5 Критерии завершения тестирования
- Выполнено ≥ 90% тест-кейсов.
- Все критические баги зарегистрированы и описаны  
- Проверен основной функционал: репозитории, коммиты, ветки, Pull Requests, Issues
- Программа соответствует требованиям SRS в основной функциональности
- Основные пользовательские сценарии выполняются без ошибок   

### 2.6 Тестовая среда
- **ОС:** GNU/Linux / MacOs 
- **Браузер:** Firefox v148  
- **Сервер:** сервер github для git actions  
- **Дополнительно:** интернет-соединение, редактор кода, утилита для снятия скриншотов
